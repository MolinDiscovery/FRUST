# Building Initial DataFrames

This tutorial shows how to create the first FRUST dataframe before running xTB,
g-xTB, ORCA, or UMA stages. The practical mental model is:

```text
SMILES or FRUST structures -> embedded conformers -> atoms + coords_embedded -> calculation stages
```

`Stepper.build_initial_df(...)` is the handoff into the dataframe model. It can
start from simple SMILES for calculator-style work, or from raw structures
generated by FRUST workflow helpers.

## Start With One Molecule

For a quick tool check, pass a SMILES string directly.

```python
from frust.stepper import Stepper

step = Stepper(save_output_dir=False)

df = step.build_initial_df("CCO", name="ethanol")
df[["substrate_name", "structure_type", "molecule_role", "cid", "smiles"]]
```

Output:

|  | substrate_name | structure_type | molecule_role | cid | smiles |
| ---: | --- | --- | --- | ---: | --- |
| 0 | ethanol | MOL | structure | 0 | CCO |

The dataframe also contains the columns that calculation stages need:

| Column | Example value | Meaning |
| --- | --- | --- |
| `atoms` | `["C", "C", "O", "H", ...]` | Element symbols for the embedded conformer |
| `coords_embedded` | `[(x, y, z), ...]` | Starting 3D coordinates |
| `input_smiles` | `CCO` | Original user input |
| `structure_id` | `MOL:ethanol:structure` | Stable structure identity |

Now run a calculation stage:

```python
df = step.gxtb(df, name="gxtb_opt", options={"opt": None})
df[["substrate_name", "cid", "gxtb_opt-NT", "gxtb_opt-EE", "gxtb_opt-oc"]]
```

!!! note "Plain SMILES means plain molecule"

    `step.build_initial_df("CCO")` means the ethanol molecule. It does not
    generate FRUST catalytic-cycle structures unless you explicitly ask for a
    workflow expansion.

## Control Conformer Generation

Raw inputs are embedded inside `build_initial_df`. The default is quick:
`n_confs=1`.

```python
df = step.build_initial_df("CCO", name="ethanol")
```

Ask for more conformers when you want conformer screening:

```python
df = step.build_initial_df("CCO", name="ethanol", n_confs=10)
df[["substrate_name", "cid", "coords_embedded"]].head()
```

Output:

|  | substrate_name | cid | coords_embedded |
| ---: | --- | ---: | --- |
| 0 | ethanol | 0 | 9 x 3 coordinates |
| 1 | ethanol | 1 | 9 x 3 coordinates |
| 2 | ethanol | 2 | 9 x 3 coordinates |

Use `n_confs=None` to use FRUST's rotatable-bond heuristic from the embedder.
For plain molecules, force-field cleanup can be requested through
`optimization`:

```python
df = step.build_initial_df(
    "CCO",
    name="ethanol",
    n_confs=10,
    optimization="UFF",
)
```

## Build Small Batches

Lists, named dictionaries, and dataframes all become the same FRUST dataframe
shape.

=== "List"

    ```python
    df = step.build_initial_df(
        ["CCO", "CCN"],
        names=["ethanol", "ethylamine"],
        n_confs=2,
    )
    ```

=== "Named Dict"

    ```python
    df = step.build_initial_df(
        {
            "ethanol": "CCO",
            "ethylamine": "CCN",
        },
        n_confs=2,
    )
    ```

=== "DataFrame"

    ```python
    import pandas as pd

    inputs = pd.DataFrame(
        {
            "substrate_name": ["ethanol", "ethylamine"],
            "smiles": ["CCO", "CCN"],
        }
    )

    df = step.build_initial_df(inputs, n_confs=2)
    ```

All three produce rows like:

| substrate_name | structure_type | molecule_role | cid | smiles |
| --- | --- | --- | ---: | --- |
| ethanol | MOL | structure | 0 | CCO |
| ethanol | MOL | structure | 1 | CCO |
| ethylamine | MOL | structure | 0 | CCN |
| ethylamine | MOL | structure | 1 | CCN |

!!! tip "Use `substrate_name` in tables"

    When building from a dataframe, prefer a `substrate_name` column for the
    human-readable molecule label. FRUST keeps the original SMILES separately
    in `smiles` and `input_smiles`.

## Use FRUST Workflow Structures

Some FRUST helpers do more than parse a molecule. For example,
`create_mol_per_rpos(...)` creates the molecule set used by the FRUST molecule
workflow. Keep that chemistry-generation step explicit, then let
`build_initial_df` embed the raw structures.

```python
import pandas as pd
from frust.stepper import Stepper
from frust.utils.mols import create_mol_per_rpos

ligands = pd.DataFrame(
    {
        "substrate_name": ["anisole"],
        "smiles": ["COc1ccccc1"],
        "rpos": ["2"],
    }
)

mols = create_mol_per_rpos(ligands, select_mols="uniques")

step = Stepper(step_type="MOLS", save_output_dir=False)
df = step.build_initial_df(mols, n_confs=2)

df[["substrate_name", "structure_type", "molecule_role", "rpos", "cid"]].head()
```

Output:

| substrate_name | structure_type | molecule_role | rpos | cid |
| --- | --- | --- | ---: | ---: |
| anisole | MOL | ligand |  | 0 |
| anisole | MOL | ligand |  | 1 |

If you want one compact call, use the explicit workflow flag:

```python
df = step.build_initial_df(
    ligands,
    workflow="mols",
    select_mols="uniques",
    n_confs=2,
)
```

!!! warning "Do not hide workflow expansion behind bare SMILES"

    `build_initial_df("COc1ccccc1")` creates plain anisole. Use
    `workflow="mols"` or `create_mol_per_rpos(...)` when you want FRUST's
    workflow molecule set.

## Use Transition-State Structures

Transition-state workflows start with a template and a substrate table.
`create_ts_per_rpos(...)` returns raw TS structures with constraint metadata.
`build_initial_df` embeds those structures and stores the constraint atoms in
the dataframe.

```python
import pandas as pd
from frust.stepper import Stepper
from frust.utils.mols import create_ts_per_rpos

ligands = pd.DataFrame(
    {
        "smiles": ["CN1C=CC=C1"],
        "rpos": ["2"],
    }
)

ts_structs = create_ts_per_rpos(
    ligands,
    ts_guess_xyz="structures/ts1.xyz",
    return_format="dict",
)

step = Stepper(step_type="auto", save_output_dir=False)
df = step.build_initial_df(ts_structs, n_confs=1, ts_optimize=False)

df[
    [
        "custom_name",
        "structure_type",
        "rpos",
        "constraint_atoms",
        "cid",
        "coords_embedded",
    ]
]
```

Output:

| custom_name | structure_type | rpos | constraint_atoms | cid | coords_embedded |
| --- | --- | ---: | --- | ---: | --- |
| `TS1(1-methylpyrrole_rpos(2))` | TS1 | 2 | `[10, 11, 39, 40, 41, 44]` | 0 | 54 x 3 coordinates |

`step_type="auto"` infers the constrained type from the dataframe. If the
dataframe contains only `TS1` rows, later constrained calculations use the
`TS1` constraint template:

```python
df = step.xtb(
    df,
    name="xtb_preopt",
    options={"gfnff": None, "opt": None},
    constraint=True,
)
```

!!! warning "Auto only chooses the constraint template"

    `step_type="auto"` does not choose a chemistry workflow. It only infers a
    single constrained type such as `TS1` or `INT3` from the dataframe built by
    `build_initial_df`.

## Already Embedded Data Still Works

Advanced users can still embed manually and pass the embedded dictionary to
`build_initial_df`. In that case, `build_initial_df` only creates dataframe
rows.

For direct embedder use, pass RDKit molecules rather than SMILES strings:

```python
from rdkit import Chem
from frust.embedder import embed_mols

raw = {"ethanol": Chem.MolFromSmiles("CCO")}
embedded = embed_mols(raw, n_confs=2)

df = step.build_initial_df(embedded)
```

## Input Summary

| Input to `build_initial_df` | Meaning | Typical use |
| --- | --- | --- |
| `"CCO"` | Plain molecule SMILES | Quick calculator checks |
| `["CCO", "CCN"]` | Batch of plain molecules | Small screens |
| `{"ethanol": "CCO"}` | Named SMILES batch | Readable labels without a dataframe |
| DataFrame with `smiles` | Batch with metadata | CSV-like workflows |
| Raw molecule dict | FRUST structures to embed | `create_mol_per_rpos(...)` output |
| Raw TS dict | TS/INT structures to embed | `create_ts_per_rpos(...)` output |
| Embedded dict | Already embedded conformers | Advanced control |

!!! note "If you already have a calculation dataframe"

    Do not send an existing calculation dataframe back through
    `build_initial_df`. Pass it directly to `step.xtb(...)`, `step.gxtb(...)`,
    or `step.orca(...)`.
